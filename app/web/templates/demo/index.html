Frontend + Backend The directory now separates demo display and API service so contributors can navigate quickly.

1. Demo display

`app/web/` only handles page routes, page context, templates, and static assets.

2. API service

`app/api/` only handles request schemas, route definitions, validation, and task response contracts.

3. Shared services

`app/services/` groups image generation, prompt building, LLM access, caching, writing orchestration, and async tasks.

4. Demo calls the API

`app/web/static/demo/demo.js` directly calls `/api/tasks` and `/api/tasks/<task_id>`, so the demo is a thin client over the live backend.
